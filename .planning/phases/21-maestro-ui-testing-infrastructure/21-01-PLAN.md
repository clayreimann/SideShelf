# Plan 01: Add testID Attributes to Login Screen, Player, and Library Elements

**Phase:** 21 -- Maestro UI Testing Infrastructure
**Goal:** Add `testID` props to all elements required by TESTING-01 and TESTING-03 so Maestro flows can target them via `id:` selectors
**Requirements:** TESTING-01, TESTING-03

## Context

This plan adds `testID` props to React Native components -- the prerequisite for all Maestro flows in later plans. Per D-03, only the specific testIDs listed in TESTING-01 and TESTING-03 are added. Per D-04, existing `testID="floating-player"` and `testID="library-item"` remain untouched.

No Maestro YAML files are created in this plan. This is purely TypeScript source changes.

## Prerequisites

None -- this is Wave 1, no dependencies on other plans.

## Tasks

### Task 1: Add testID attributes to login screen inputs (TESTING-01)

**File(s):** `src/app/login.tsx`

**Action:** Add `testID` props to the four interactive elements on the login screen:

1. Server URL `TextInput` -- add `testID="login-server-url-input"`
2. Username `TextInput` -- add `testID="login-username-input"`
3. Password `TextInput` -- add `testID="login-password-input"`
4. Login `TouchableOpacity` button -- add `testID="login-button"`

Per D-03, these are the only testIDs needed on the login screen. Do not add testIDs to any other elements.

**Details:**

- Open `src/app/login.tsx`
- Locate the three `TextInput` elements (server URL around line 119, username around line 129, password around line 138)
- Add the `testID` prop to each
- Locate the login `TouchableOpacity` (around line 155) and add `testID="login-button"`
- Run `npm test -- --findRelatedTests src/app/login.tsx` to confirm no test breakage (login.tsx is in `src/app/` which is excluded from coverage, but related tests may exist)

### Task 2: Add testID attributes to key interactive elements (TESTING-03)

**File(s):**

- `src/components/player/ConsolidatedPlayerControls.tsx`
- `src/components/player/PlayPauseButton.tsx`
- `src/app/FullScreenPlayer/index.tsx`
- `src/components/player/ProgressBar.tsx`
- `src/components/player/PlaybackSpeedControl.tsx`
- `src/components/library/LibraryItemDetail.tsx`
- `src/components/library/LibraryItemList.tsx`

**Action:** Add `testID` props to the six elements specified in TESTING-03 (per D-03):

1. **`play-resume-button`** in `ConsolidatedPlayerControls.tsx`: Add `testID="play-resume-button"` to the green standalone "Play" `TouchableOpacity` (the not-playing branch, around line 148). Do NOT add this testID to `PlayPauseButton.tsx` directly -- that would create duplicate testIDs on screen (see Research Pitfall 6). Instead, add a `testID?: string` prop to `PlayPauseButton`'s interface and pass it through to the inner `Pressable`. The FullScreenPlayer can pass `testID="play-pause-button"` if needed in future, but for this task only the `ConsolidatedPlayerControls` standalone play button gets `testID="play-resume-button"`.

2. **`player-done-button`** in `FullScreenPlayer/index.tsx`: Add `testID="player-done-button"` to the chevron-down `TouchableOpacity` that dismisses the full screen player (around line 531). The existing `capture-screenshots.yaml` taps `"Done"` by text which still works -- this testID provides explicit targeting for new flows.

3. **`seek-slider`** in `ProgressBar.tsx`: Add a `testID?: string` prop to the `ProgressBarProps` interface. Pass it to the interactive `View` wrapper (the `paddingVertical: 8` view with `{...panResponder.panHandlers}` around line 196). The `FullScreenPlayer` renders `ProgressBar` with `interactive={true}` -- add `testID="seek-slider"` at that call site in `FullScreenPlayer/index.tsx`.

4. **`speed-control`** in `PlaybackSpeedControl.tsx`: Add `testID="speed-control"` to the inner `View` element (the speed badge container, around line 43).

5. **`download-button`** in `LibraryItemDetail.tsx`: Add `testID="download-button"` to the `View` wrapping the ellipsis icon inside the `MenuView` (around line 623). This is the tap target that opens the native menu. Per Research Pitfall 4, Maestro will tap this testID to open the menu, then tap "Download" by text.

6. **`library-search-input`** in `LibraryItemList.tsx`: Add `testID="library-search-input"` to the `TextInput` in the `ListHeaderComponent` (around line 45).

**Details:**

- For `PlayPauseButton.tsx`: add `testID?: string` to the component props, pass through to the `Pressable`. Callers not passing `testID` get no testID (preserves existing behavior).
- For `ProgressBar.tsx`: add `testID?: string` to `ProgressBarProps`, pass to the interactive wrapper `View` only. Non-interactive progress bars are not affected.
- Run `npm test` to confirm no test breakage after all changes.
