**Step 3 — Rewrite CollapsibleSection.tsx** implementing the architecture from RESEARCH.md:

```
CollapsibleSection
├── Outer wrapper View (marginBottom: 16)
│
├── [If title provided] Header Pressable (header-only toggle)
│   ├── Text (title, 16px / fontWeight: '500', colors.textPrimary)
│   └── Animated.View (Ionicons chevron-forward 16px, rotates 0°↔90° via withTiming)
│
├── [If no title AND isCollapsible] Pressable wraps content below
│   (DescriptionSection tap-anywhere model)
│
├── [Before layoutMeasured] plain View — children render freely for onLayout measurement
│   View onLayout → captures height → sets layoutMeasured = true
│
├── [After layoutMeasured AND isCollapsible] Animated.View (height: animated, overflow: hidden)
│   ├── View (onLayout — measures full height; only first measurement used)
│   │   └── children (always mounted — no conditional rendering)
│   └── Animated.View (testID="collapsible-gradient", position: absolute, bottom:0, pointerEvents="none")
│       └── LinearGradient (colors=['transparent', colors.background], start={0,0} end={0,1})
│
├── [After layoutMeasured AND NOT isCollapsible] plain View — children shown as-is
```

**Constants:**

```typescript
const PEEK_HEIGHT = 100;
const ANIMATION_DURATION = 280; // ~250–300ms per CONTEXT.md discretion
const GRADIENT_HEIGHT = 48; // fixed gradient height — Claude's discretion
```

**Shared values:**

```typescript
const measuredHeight = useSharedValue(0);
const isExpandedSV = useSharedValue(defaultExpanded ? 1 : 0);
const gradientOpacity = useSharedValue(defaultExpanded ? 0 : 1);
```

**Toggle handler:**

```typescript
const toggle = () => {
  const expanding = isExpandedSV.value === 0;
  isExpandedSV.value = expanding ? 1 : 0;
  gradientOpacity.value = expanding ? 0 : 1;
};
```

**Animated styles:**

- `containerStyle`: height animates between PEEK_HEIGHT and measuredHeight.value using withTiming(duration: ANIMATION_DURATION)
- `gradientStyle`: opacity animates via withTiming on gradientOpacity
- `chevronStyle`: rotate withTiming '0deg' ↔ '90deg'

**onLayout guard:** Only capture the first measurement (`if (layoutMeasured) return`). Set `isCollapsible = true` only if measured height > PEEK_HEIGHT.

**DescriptionSection compatibility:** The call site currently passes `title={translate("libraryItem.description")} defaultExpanded={true}`. The new API makes `title` optional. DescriptionSection will be updated in Plan 03 to remove the title prop. The component must work correctly both with and without title.

**Pitfalls to avoid (from RESEARCH.md):**

- Do NOT use `{isExpanded && children}` — always mount children
- Do NOT use `useNativeDriver: false` with RN core Animated — use Reanimated only
- Do NOT set gradient `pointerEvents` to anything other than `"none"` — it must not block touches
- Do NOT animate height before layoutMeasured is true — use two-phase render
- Do NOT use the `useDerivedValue` multiplication trick (multiplying by boolean 0 loses peek state)
- Gradient colors: `['transparent', colors.background]` — top transparent, bottom = screen background
