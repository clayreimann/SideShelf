**Part A — WAL pragma in client.ts:**

In `getSQLiteDb()`, immediately after `sqliteDb = SQLite.openDatabaseSync(DB_NAME)` and before the closing brace, add two separate `execSync` calls:

```typescript
sqliteDb.execSync("PRAGMA journal_mode=WAL;");
sqliteDb.execSync("PRAGMA synchronous=NORMAL;");
```

Two separate calls (not one combined string) as specified in RESEARCH.md. Add JSDoc comment explaining: WAL is persistent (set once, persists across opens); synchronous=NORMAL is connection-level (must be set each open, getSQLiteDb handles this via the if (!sqliteDb) guard).

Do NOT use `db.run(sql\`PRAGMA\`...)`— must use raw`sqliteDb.execSync()` before Drizzle wraps the connection.

**Part B — DbErrorScreen component:**

Create `src/components/errors/DbErrorScreen.tsx`. Props: `{ error: Error; onReset?: () => void }`.

The component should:

- Display a user-friendly heading: "Database Error"
- Display a generic message: "Something went wrong with the database. The app cannot continue."
- Show a scrollable technical error block with `error.message` and `error.stack` (if available)
- Show a "Copy Error Details" button that calls `Share.share({ message: \`${error.message}\n\n${error.stack ?? ''}\` })`from`react-native`
- Detect error category by checking `error.message.toLowerCase()` for:
  - Contains "no space" or "disk full" or "enospc": show category message "Your device storage is full. Free up space and restart the app." — do NOT show reset button (resetting won't help)
  - All other errors: show "Reset Database" button that calls `onReset` with a warning "Warning: this will delete all locally cached data" — data on the server is not affected
- Use basic `View`/`Text`/`Pressable`/`ScrollView` from `react-native` — no custom UI components needed, plain styles are fine
- Log the error on mount using `logger.error('DbProvider', 'DB error screen shown', error)` — this writes to the separate `logs.sqlite`, not the failing `abs2.sqlite`
