<feature>
  <name>progressFormat utility</name>
  <files>src/lib/helpers/progressFormat.ts, src/lib/helpers/__tests__/progressFormat.test.ts</files>
  <behavior>
    Type:
    - ProgressFormat = "remaining" | "elapsed" | "percent"

    formatProgress(format: ProgressFormat, positionSecs: number, durationSecs: number): string
    - "remaining": returns "${Xh Ym} remaining" using friendly units (no seconds shown). Example: position=5025, duration=13530 → "2h 20m remaining". Uses formatTimeWithUnits-style logic: hours + minutes only, no seconds suffix.
    - "elapsed": returns "H:MM:SS / H:MM:SS" clock format. Example: position=5025, duration=13530 → "1:23:45 / 3:45:30"
    - "percent": returns "${N}%" rounded to nearest integer. Example: position=5025, duration=13530 → "37%"
    Edge cases:
    - duration=0: "remaining" → "0m remaining", "elapsed" → "0:00 / 0:00", "percent" → "0%"
    - position=0: "remaining" → full duration remaining, "elapsed" → "0:00 / H:MM:SS", "percent" → "0%"
    - position >= duration: "remaining" → "0m remaining", "percent" → "100%"

    The formatTimeWithUnits helper (currently private in FullScreenPlayer) must be extracted here — no duplication. The helper for "remaining" shows hours + minutes only (no seconds), consistent with the hardcoded text being replaced.

  </behavior>
  <implementation>
    After tests pass:
    - Export `ProgressFormat` as a string literal union type
    - Export `formatProgress(format, positionSecs, durationSecs): string`
    - Internal helper `formatClock(seconds: number): string` for H:MM:SS / MM:SS clock format used in "elapsed"
    - Internal helper `formatFriendlyDuration(seconds: number): string` for "Xh Ym" without seconds, used in "remaining"
    - No external imports — pure function, no React, no store
    - File location: `src/lib/helpers/progressFormat.ts` (alongside existing `formatters.ts`)
  </implementation>
</feature>
