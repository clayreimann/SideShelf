## Decisions Made

- DispatchMeta is a separate type, not merged into PlayerEvent union variants — keeps discriminated union clean; coordinator reads meta from bus side-channel not event payload
- `meta` stored in eventHistory entries (`{ event, meta?, timestamp }`) so coordinator diagnostics can see dispatch origin
- `source.db` span only opened when memory check found no `libraryItemId` — avoids empty DB span on warm restores
- Early return paths (no_username, user_not_found, no_library_item_id, no_session) all end rootSpan with `earlyExit` attribute before returning
