## Decisions Made

- `expo-application` mock added globally to `setup.ts` with `{ virtual: true }` — the integration tests pull in `PlayerStateCoordinator` transitively and would fail without it; a per-file mock would require touching every integration test
- `source` accessed via `(event as any).source` cast with eslint-disable comment — the field is not in the `PlayerEvent` union type yet (Plan 03 adds it at the event bus level); a follow-up refactor will thread it cleanly through the type system
- `writeDumpToDisk` is never awaited inside `handleEvent` — it's called fire-and-forget with `.catch()` to comply with the AsyncLock pitfall (waiting inside the lock would block the queue)
