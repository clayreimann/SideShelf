<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
    Complete bookmark management UX:
    - First-tap preference alert (Auto-create vs Always Prompt)
    - Auto-create mode: title = chapter title + em dash + timestamp
    - Prompt mode: input pre-filled with auto-title, user can edit
    - Long-press BookmarkButton in auto mode triggers one-time title override
    - BookmarksSection in item detail: clean rows (no trash icon), long-press opens context menu
    - Rename bottom sheet with TextInput pre-filled with current title
    - Delete via context menu (no secondary confirmation on iOS, Alert confirm on Android)
    - Tap bookmark → seeks player to that position
    - Settings → Player section → Bookmark Title Mode row
    - SQLite offline cache: bookmarks visible after server disconnected
  </what-built>
  <how-to-verify>
    Run: `npx expo start` and open on iOS simulator (or device).

    **Flow 1 — First-tap + auto mode:**
    1. Open any item in the library that has active playback (or start playback)
    2. Open the full-screen player
    3. Tap the bookmark button → "Bookmark Style" alert should appear with "Auto-create" and "Always Prompt"
    4. Tap "Auto-create"
    5. Bookmark created alert appears (or silent success) — verify no error
    6. Navigate to the item detail screen → Bookmarks section → bookmark row visible with chapter+timestamp title

    **Flow 2 — Auto mode second tap:**
    1. Tap bookmark button again — no alert this time; bookmark created immediately
    2. Verify bookmark appears in item detail

    **Flow 3 — Rename:**
    1. On item detail, long-press a bookmark row
    2. iOS ActionSheet should appear with "Rename" and "Delete" options
    3. Tap "Rename" → bottom sheet slides up with TextInput pre-filled
    4. Edit the title and tap "Save" → row updates in the list

    **Flow 4 — Delete:**
    1. Long-press a bookmark row → ActionSheet
    2. Tap "Delete" (red) → bookmark removed from list (no trash icon visible in rows)

    **Flow 5 — Seek:**
    1. Tap any bookmark row (not long-press) → player seeks to that timestamp
    2. Verify position indicator moves to bookmark time

    **Flow 6 — Settings row:**
    1. More → Settings → Player section → "Bookmark Title Mode" row visible
    2. Tap it → Alert with "Auto-create" and "Always Prompt" options
    3. Select "Always Prompt"

    **Flow 7 — Prompt mode:**
    1. Return to full-screen player, tap bookmark button
    2. Alert.prompt appears (iOS) pre-filled with chapter+timestamp title
    3. Edit title, tap OK → bookmark created with custom title

    **Flow 8 — Long-press override in auto mode:**
    1. Change back to Auto-create in Settings
    2. Long-press the bookmark button (not tap) → prompt input appears
    3. Enter title → bookmark created with custom title despite auto mode

    **Flow 9 — SQLite persistence:**
    1. Restart the app (Cmd+Shift+D → Reload in Expo, or kill and reopen)
    2. Navigate to the item detail → bookmarks still visible (loaded from SQLite)

  </how-to-verify>
