    **Create `src/db/migrations/0014_normalize_paths.sql`:**
    Write the SQL UPDATE statements as three passes (per RESEARCH.md Pattern 4):

    Pass 1 — Strip file:// prefix (all three tables):
    ```sql
    -- Strip file:// prefix from local_audio_file_downloads
    UPDATE local_audio_file_downloads
    SET download_path = SUBSTR(download_path, 8)
    WHERE download_path LIKE 'file://%';

    -- Strip file:// prefix from local_library_file_downloads
    UPDATE local_library_file_downloads
    SET download_path = SUBSTR(download_path, 8)
    WHERE download_path LIKE 'file://%';

    -- Strip file:// prefix from local_cover_cache
    UPDATE local_cover_cache
    SET local_cover_url = SUBSTR(local_cover_url, 8)
    WHERE local_cover_url LIKE 'file://%';
    ```

    Pass 2 — Decode common percent-encoded characters (all three tables):
    ```sql
    -- Decode %20 (space), %28 ((), %29 ()) in download paths
    UPDATE local_audio_file_downloads
    SET download_path = REPLACE(REPLACE(REPLACE(download_path, '%20', ' '), '%28', '('), '%29', ')')
    WHERE download_path LIKE '%\%%' ESCAPE '\';

    UPDATE local_library_file_downloads
    SET download_path = REPLACE(REPLACE(REPLACE(download_path, '%20', ' '), '%28', '('), '%29', ')')
    WHERE download_path LIKE '%\%%' ESCAPE '\';

    UPDATE local_cover_cache
    SET local_cover_url = REPLACE(REPLACE(REPLACE(local_cover_url, '%20', ' '), '%28', '('), '%29', ')')
    WHERE local_cover_url LIKE '%\%%' ESCAPE '\';
    ```

    IMPORTANT (from RESEARCH.md): Do NOT attempt to convert absolute paths to D:/C: prefix in SQL — `toAppRelativePath()` uses runtime `Paths.document.uri` and `Paths.cache.uri` which are device-dynamic. The `resolveAppPath()` fallback already handles legacy bare absolute paths at read time.
