    **Critical pitfall (from RESEARCH.md Pitfall 2):** Generate `restoreSessionId` as a local variable at the top of the method (`const restoreSessionId = Math.random().toString(16).slice(2);`) — do NOT use a module-level variable.

    Instrument `restorePlayerServiceFromSession()` as follows:

    **1. Root span** — wrap the entire method:
    ```typescript
    const restoreSessionId = Math.random().toString(16).slice(2);
    const rootSpan = trace.startSpan('player.restore.session', { restoreSessionId });
    try {
      // ... existing method body ...
      trace.endSpan(rootSpan, 'ok', { restoreSessionId });
    } catch (error) {
      trace.recordError(error, rootSpan);
      trace.endSpan(rootSpan, 'error');
      throw error;
    }
    ```

    **2. Source spans** — wrap each data source query in a child span with explicit parentContext:
    - `player.restore.source.memory` — wraps the store check (`store.player.currentTrack`): record `{ found: !!libraryItemId, itemId: libraryItemId ?? null }`
    - `player.restore.source.db` — wraps the `getAllActiveSessionsForUser` call: record `{ found: sessions.length > 0, sessionCount: sessions.length, itemId: chosenItemId ?? null }`
    - `player.restore.source.track_player` — wraps the TrackPlayer state/queue check (if one exists in the method): record `{ found, state, queueIndex }`

    Each source span:
    ```typescript
    const dbSpan = trace.startSpan('player.restore.source.db', {}, rootSpan.context);
    // ... query ...
    trace.endSpan(dbSpan, 'ok', { found: true, itemId, positionMs, sessionId });
    ```

    **3. Reconcile span** — wrap candidate selection logic:
    ```typescript
    const reconcileSpan = trace.startSpan('player.restore.reconcile', {}, rootSpan.context);

    trace.addEvent('restore.decision.finalized', { source: 'db', positionMs, restoreSessionId });

    ```

    **4. Apply span** — wrap the track/queue rebuild:

    ```

    **5. Verify span** — wrap the final state check (if one exists; otherwise emit a single addEvent):

    ```
