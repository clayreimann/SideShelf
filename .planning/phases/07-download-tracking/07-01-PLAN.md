Add a new exported async function `runDownloadReconciliationScan(): Promise<void>` to `src/lib/fileLifecycleManager.ts`.

The function must:

1. **Stale record scan** — iterate `getDownloadedAudioFilesWithLibraryInfo()`, skip any item whose `libraryItemId` passes `downloadService.isDownloadActive(libraryItemId)` (covers both actively-transferring AND paused downloads), call `verifyFileExists(downloadInfo.downloadPath)` on each non-skipped file, and call `clearAudioFileDownloadStatus(downloadInfo.audioFileId)` for any file that does not exist on disk. 2. **Zustand store invalidation** — after processing all stale records, call `useAppStore.getState().removeDownloadedItem(libraryItemId)` for each library item that had at least one file cleared. For each returned `task`, extract `libraryItemId` from `task.id` (the task ID format is `${libraryItemId}-${audioFileId}` — split on first '-' to get libraryItemId, or inspect the existing `restoreExistingDownloads` method in `DownloadService.ts` to see the exact ID format and replicate). If `downloadService.isDownloadActive(libraryItemId)` returns false AND the task is not done/paused, it is a zombie: call `task.stop()`, then delete the item's download directory using `getDownloadsDirectory(libraryItemId, 'documents').delete()` and `getDownloadsDirectory(libraryItemId, 'caches').delete()` (check `.exists` before deleting), then call `clearAudioFileDownloadStatus` for each audio file in that item (use `getDownloadedAudioFilesWithLibraryInfo()` result filtered by `libraryItemId`, or `getAudioFilesWithDownloadInfo` via the metadata helper). Log: `log.warn('[ReconciliationScan] Zombie cleared: ${libraryItemId}')`.

- `import { downloadService } from '@/services/DownloadService';`
- `import RNBackgroundDownloader from '@kesha-antonov/react-native-background-downloader';`
- `import { getDownloadsDirectory } from '@/lib/fileSystem';` (already imported via fileSystem, but add `getDownloadsDirectory` to the import list)

Do NOT call `clearAudioFileDownloadStatus` on any file whose `libraryItemId` is in `activeDownloads`. The `isDownloadActive` check is the only guard needed — it covers both active and paused states.

Do NOT modify `detectCleanedUpFiles()` — create a new `runDownloadReconciliationScan()` function alongside it. The existing function can remain for backward compatibility.

```typescript
removeDownloadedItem: (libraryItemId: string) => {
  log.info(`[DownloadSlice] Removing download record for ${libraryItemId} (reconciliation scan)`);
  set((state: DownloadSlice) => {
    const newDownloadedItems = new Set(state.downloads.downloadedItems);
    newDownloadedItems.delete(libraryItemId);
    const newPartiallyDownloadedItems = new Set(state.downloads.partiallyDownloadedItems);
    newPartiallyDownloadedItems.delete(libraryItemId);
    return {
      ...state,
      downloads: {
        ...state.downloads,
        downloadedItems: newDownloadedItems,
        partiallyDownloadedItems: newPartiallyDownloadedItems,
      },
    };
  });
},
```
